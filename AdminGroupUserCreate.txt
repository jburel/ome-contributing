Admin: Group/User Create
==========================

**To test in**: Web, Insight

**Purpose of the test** is to check the creation of new groups/users functionality when logged in as administrator

#. Go to administration tab (left pane harmonica (Insight) or Admin tab (Web) [:ref:`AdminGrUsCrScreen001`]), 

#. Create new group 

   - using the icon above the tree [:ref:`AdminGrUsCrScreen002`] (Insight only)
   - using the right-click (Insight only)
   - using the button (called "Add new..." in Web, Web only) [:ref:`AdminGrUsCrScreen003`][:ref:`AdminGrUsCrScreen004`]  
   - create private/read-only/read-annotate group
   - |C| new group was created with permissions as chosen at creation

#. Create new user [:ref:`AdminGrUsCrScreen005`][:ref:`AdminGrUsCrScreen006`][:ref:`AdminGrUsCrScreen007`]

   - tick/do not tick the Administrator box in the creation window
   - |C| that the new user is (or is not) an Administrator as appropriate
   - tick/do not tick the Group owner box (Insight only) [:ref:`AdminGrUsCrScreen008`]
   - |C| that the new user is (or is not) Group owner as appropriate
   - add the new user to one or more existing groups [:ref:`AdminGrUsCrScreen009`]
   - |C| that the user has been added to the groups correctly
