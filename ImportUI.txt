Import (UI)
===========



**To test in**: Insight

**Purpose of the test** is to check all the features of Importer UI as started from Insight

Find suitable images here [:download:`TestingSet </downloads/ImportTestingSetTable.pdf>`] or here [:download:`TestingTemplate </downloads/ImportTestingTemplate.pdf>`]

#. Select single image files

   - Add them to the queue by pressing the ">" control.
   - Select "New From Folder" option [:ref:`ImportUI001`] when choosing where to import the images.
   - |C| that a new Dataset will be created with the name of the folder.
   - |C| that the icon of the Dataset (right-hand side of the Import window) is light blue, not plum color as for Project [:ref:`ImportUI002`]
   - Click import.
   - Go to the DataManager.
 
     - |C| that the wheel in the toolbar is spinning when import in progress [:ref:`ImportUI004`]
     - |C| that the number next to green tick is incremented when an import is finished successfully.

   - Still in Data Manager (left-hand side), hit the refresh button
     - |C| that only one dataset is created. [:ref:`ImportUI003`]

#. Repeat (1) this time selecting a whole folder to import. [:ref:`ImportUI004`].

   - |C| that the counter in toolbar of Data Manager (right-hand side) counts the imported files correctly [:ref:`ImportUIgreen tick and wheel`].

#. Repeat (1) this time not selecting "New From Folder" but creating a new Project and Dataset at import stage [:ref:`ImportUIcreate project`, :ref:`ImportUIcreate dataset`].

#. Tag at import (for permissions testing, tag with tags belonging to other users)

   - Select at least 2 images.
   - Go to the "options" tab. The second tab when selecting the import location.[:ref:`ImportUI005`]
   - Create a new tag
   - Import the images.
   - Go to the DataManager (left-hand side). Refresh
   - |C| that the images are tagged and the tag is shown on thumbnails [:ref:`ImportUI006`][:ref:`ImportUI007`]
   - |C| and the tag is created once. [:ref:`ImportUI008`]

#. Repeat (3) and this time (for permissions testing, tag with tags belonging to other users)

   - create a new tag and add an existing tag.[:ref:`ImportUI009`]
   - Import the images
   - Go to the DataManager (left-hand side). Refresh,
   - |C| that the images are tagged and the tag is shown on thumbnails
   - |C| that the tag is created once
   - |C| that no new tag is created with the name of an existing tag.

#. In the Location dialog, press the Screen button [:ref:`ImportUI011`]

#. Import a Plate e.g. ``test_images_good/bd-pathway/2009-05-01_000/`` 

   - not creating new Screen at import stage [:ref:`ImportUIno new screen`]
   - creating a new Screen to import the plate to at import stage [:ref:`ImportUI011new screen`]



#. When the import is done, check in the import Window:

   - |C| that thumbnail(s) is(are) created. Note that the "Show thumbnail" must have been selected prior to the import. [:ref:`ImportUI012`]
   - |C| that when you hover over thumbnail(s), a larger thumbnail view pops up [:ref:`ImportUI013`].
   - |C| that click on the thumbnail(s) opens the image in Viewer. In case of a multi-image file or plate [:ref:`ImportUIclick on thumbnail`], click on every thumbnail in the row.
   - |C| that a click onto the blue link brings you to the dataset in Data Manager [:ref:`ImportUIblue link`].

#. Import an image that will fail to import e.g. ``data_repo/test_images_bad/mike1_R3D.dv`` [:ref:`ImportUI014`]

   - |C| that you can view the error.[:ref:`ImportUI015`]
   - Submit the error to the QA system using "Submit Failures" button in the right bottom corner [:ref:`ImportUIsubmit failures`]. 
   - Enter a valid e-mail address [:ref:`ImportUI016`]
   - |C| that you received a notification e-mail.
   - |C| that your email address, error message, and file links are all included on the QA webpage.
   - Send a reply comment to the QA webpage, include your email address and a message.
   - Log into the QA system as an administrator and confirm the comment was received.
   - Reply to the QA message, the message should appear in your email.
   - Finally, set the QA message status to "Closed" and save.
