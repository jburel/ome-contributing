Import (UI)
===========



**To test in**: Insight

**Purpose of the test** is to check all the features of Importer UI as started from Insight

Find suitable images here [:download:`TestingSet </downloads/ImportTestingSetTable.pdf>`] or here [:download:`TestingTemplate </downloads/ImportTestingTemplate.pdf>`]

#. Select single image files

   - Add them to the queue by pressing the ">" control.
   - Select "New From Folder" option [:ref:`ImportUIScreen001`] when choosing where to import the images.
   - |C| A new Dataset will be created with the name of the folder.
   - |C| Make sure the icon of the Dataset (right-hand side of the Import window) is light blue, not plum color as for Project [:ref:`ImportUIScreen002`]
   - Click import.
   - Go to the DataManager (left-hand side). Hit the refresh button
   - |C| make sure that only one dataset is created. [:ref:`ImportUIScreen003`]

#. Repeat (1) this time by selecting a folder. [:ref:`ImportUIScreen004`]

#. Tag at import

   - Select at least 2 images.
   - Go to the "options" tab. The second tab when selecting the import location.[:ref:`ImportUIScreen005`]
   - Create a new tag
   - Import the images.
   - Go to the DataManager (left-hand side). Refresh
   - |C| that the images are tagged and the tag is shown on thumbnails [:ref:`ImportUIScreen006`][:ref:`ImportUIScreen007`]
   - |C| and the tag is created once. [:ref:`ImportUIScreen008`]

#. Repeat (3) and this time

   - create a new tag and add an existing tag.[:ref:`ImportUIScreen009`]
   - Import the images
   - Go to the DataManager (left-hand side). Refresh,
   - |C| that the images are tagged and the tag is shown on thumbnails
   - |C| The tag is created once
   - |C| no new tag is created with the name of the existing tag.

#. In the Location dialog, press the Screen button [:ref:`ImportUIScreen011`]

#. Import a Plate e.g. ``data_repo/from_skyking/flex/achim/003003000.flex``

#. When the import is done, check in the import Window:

   - |C| If thumbnail is created. Note that the "Show thumbnail" must have been selected prior to the import. [:ref:`ImportUIScreen012`]
   - |C| Hover over it.[:ref:`ImportUIScreen013`]
   - |C| Click on the thumbnail to open the image viewer
   - |C| If Dataset is highlighted in blue, click on the dataset. This should bring you to the Data Manager.

#. Import an image that will fail to import e.g. ``data_repo/test_images_bad/mike1_R3D.dv`` [:ref:`ImportUIScreen014`]

   - |C| that you can view the error.[:ref:`ImportUIScreen015`]
   - Submit the error to the QA system using "Submit Failures" button in the right bottom corner. [:ref:`ImportUIScreen016`]
   - Enter a valid e-mail address
   - |C| that you received a notification e-mail.
   - |C| that your email address, error message, and file links are all included on the QA webpage.
   - Send a reply comment to the QA webpage, include your email address and a message.
   - Log into the QA system as an administrator and confirm the comment was received.
   - Reply to the QA message, the message should appear in your email.
   - Finally, set the QA message status to "Closed" and save.
